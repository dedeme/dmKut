Características

Las marcas de bloque comienzan en un párrafo cuyo único contenido es la propia marca y se extienden hasta un párrafo que comienze con }>.

En el bloque puede haber varias secciones.

Cada sección empieza con una línea sin espacios en blanco iniciales y continúa, si es preciso, con más líneas que comienzan, al menos con un blanco.

Entre las secciones se pueden poner tantas lineas en blanco como se desee.

Lista

Se corresponde con las marcas html <ol> y <ul>.

Marca: <{l[1|a]?

Opciones:

1
Crea una lista numerada de tipo 1., 2., 3. ...
a
Crea una lista numerada de tipo a., b., c. ...
?
Si no se indica el tipo de numeración, la lista será 'no ordenada'.

Ejemplos:

Una lista numerada: <{l1 Primera entrada. Segunda entrada con dos líneas en el diseño; pero una sola en el resultado. Tercera entrada una linea en el diseño;\\\\pero dos en el resultado. }> Otra lista numerada: <{la Primera entrada. Segunda entrada con dos líneas en el diseño; pero una sola en el resultado. Tercera entrada una linea en el diseño;\\\\pero dos en el resultado. }> Finalmente una lista 'no-numerada': <{l Primera entrada. Segunda entrada con dos líneas en el diseño; pero una sola en el resultado. Tercera entrada una linea en el diseño;\\\\pero dos en el resultado. }>

muestra

Una lista numerada:

  1. Primera entrada.
  2. Segunda entrada con dos líneas en el diseño; pero una sola en el resultado.
  3. Tercera entrada una linea en el diseño;
    pero dos en el resultado.

Otra lista numerada:

  1. Primera entrada.
  2. Segunda entrada con dos líneas en el diseño; pero una sola en el resultado.
  3. Tercera entrada una linea en el diseño;
    pero dos en el resultado.

Finalmente una lista 'no-numerada':

  • Primera entrada.
  • Segunda entrada con dos líneas en el diseño; pero una sola en el resultado.
  • Tercera entrada una linea en el diseño;
    pero dos en el resultado.

Definición

Se corresponde con la marca html <dl> .

Marca: <{d

Ejemplo:

Unas definiciones: <{d //Término 1// Una definición en una línea __Término 2__ Otra definición con dos líneas en el diseño; pero una sola en el resultado. Término 3 Tercera definición con una linea en el diseño;\\\\pero dos en el resultado. }>

muestra

Unas definiciones:

Término 1
Una definición en una línea
Término 2
Otra definición con dos líneas en el diseño; pero una sola en el resultado.
Término 3
Tercera definición con una linea en el diseño;
pero dos en el resultado.

Tabla

Sólo se admiten tablas regulares de tipo filas x columnas.

El número de columnas se indica en la marca inicial, al definir su alineación.

Las celdas se irán rellenando con los contenidos de cada sección. Cuando se agote una fila, se empezará con la sigiente. Si quedan posiciones vacías en la última línea, se completarán con las correspondientes <td></td>.

Una celda se puede dejar vacía escribiendo en su sección un signo '-' sin más.

Marca: {[T|t]|?[l|c|r]+

Opciones:

T
Crea una table centrada.
t
Crea una tabla con alineación a la izquierda.
|
(Opcional) La tabla tendrá lineas de separación de celdas.
l
Alineación de columna a la izquierda.
c
Alineación de columna en el centro.
r
Alineación de columna a la derecha.
+
Se añadirán tantas marcas de alineación como columnas tenga la tabla.
Al menos se indicará una.

Ejemplos:

Una Tabla a la derecha: <{trl 1:·· Se escribe "uno" 2:·· Se escribe\\\\"dos" y···\\\\3:·· Se escribe "tres }> Y una tabla centrada con líneas: <{T|rl 1:·· Se escribe "uno" 2:·· Se escribe\\\\"dos" y···\\\\3:·· Se escribe "tres }>

muestra

Una Tabla a la derecha:

1:   Se escribe "uno"
2:   Se escribe
"dos"
y   
3:  		 Se escribe "tres

Y una tabla centrada con líneas:

1:   Se escribe "uno"
2:   Se escribe
"dos"
y   
3:  		 Se escribe "tres

Código

Inserta un recuadro mara mostrar código correpondiente a lenguajes de programación y similares.

Esta marca funciona en conexión con el encabezado general y supone tener configurado el editor ACE.

Marca: <{[C|c](0-9) label

Opciones:

C
Muestra el recuadro centrado.
c
Muestra el recuadro a la izquierda.
0-9
Indica uno de los diez posibles lenguages configurados en el encabezado general.
label
Una etiqueta que debe ser diferente para cada recuadro.

Ejemplos:

En los ejemplos siguientes se supone que en el encabezado principal dentro de la función aceConf se han definido los siguientes lenguajes:

function aceConf(ed, langIx) { // ... switch (langIx) { case 0: ed.getSession().setMode("ace/mode/kmarkdown"); break; case 1: ed.getSession().setMode("ace/mode/html"); break; case 2: ed.getSession().setMode("ace/mode/javascript"); break; default: console.log("Ace language " + langIx + " not found."); } // ... }

 

Ejemplo de código en C. <{c2 ed1 function printSomeThing(s) { if (s.length > 10) { console.log("This is \n" + s); } } }> Ejemplo de código HTML centrado <{C1 ed2 <p>Un párrafo con <b>negrita</b>. }>

El resultado es:

Ejemplo de código en C.

function printSomeThing(s) { if (s.length > 10) { console.log("This is \n" + s); } }

Ejemplo de código HTML centrado

<p>Un párrafo con <b>negrita</b>.</p>

Texto html literal

Permite introducir largos bloques de html directamente.

Marca: <{V

Ejemplo:

<{V <p>Una tabla numerando con letras mayúsculas.</p> <ol type="A"> <li>Primera entrada.</li> <li>Segunda entrada.</li> <li>Trecera entrada.</li> </ol> }>

mostrará

Una tabla numerando con letras mayúsculas.

  1. Primera entrada.
  2. Segunda entrada.
  3. Trecera entrada.
